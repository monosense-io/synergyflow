# ADR 0002 — UUIDv7 for Primary Keys

Date: 2025-10-06
Status: Accepted

## Context
UUIDv4 causes random B‑tree inserts and poor locality in PostgreSQL. RFC 9562 introduced UUIDv7 (time‑ordered).

## Decision
Use UUIDv7 for primary keys (Hibernate 7 `@UuidGenerator(style = VERSION_7)`), or custom generator fallback prior to upgrade.

## Consequences
- Pros: Better index locality, faster inserts, chronological ordering.
- Cons: Requires ORM/version alignment; migration planning if upgrading from 6.x.
